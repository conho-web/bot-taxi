# bot-taxi — VK-бот для анонимных заказов такси

Node.js + [Callback API](https://dev.vk.com/api/callback-api) ВКонтакте + SQLite.

## Возможности

- **Пассажир** — в ЛС: **«🚕 Заказать такси»** → маршрут. После ответа водителя — **«Да, едем»** / **Отмена**; в поездке переписка с водителем **в этом же диалоге с ботом**. Завершить поездку может **только водитель**.
- **Водитель** — в ЛС: **«🚗 Я водитель»** и позывной → заявка админу. После одобрения заказы в **беседе водителей** (кнопки **~ 3-5 / ~ 10 / ~ 20 мин**). Переписка и **«🏁 Завершить заказ»** — в личке с ботом.
- **Цены** — кнопка **«💰 Цены»**; текст в файле `prices.txt` (можно менять на сервере или через админку).
- **Админ** — в `.env` задаётся `ADMIN_VK_IDS`; в ЛС кнопка **«⚙️ Админ»** или `/admin`: одобрение водителей, смена цен.
- По нажатию кнопки в беседе пассажир получает предложение в ЛС с кнопками **Да, едем** / **Отмена**.
- После подтверждения водитель и пассажир пишут друг другу **только в личных сообщениях бота** (не в беседе водителей).
- Свободные сообщения в беседе водителей **пассажирам не пересылаются**.

## Настройка ВК

1. Создайте **сообщество**, включите **Сообщения сообщества**.
2. **Управление → Работа с API**: создайте ключ с доступом к сообщениям (и при необходимости к управлению сообществом по документации VK).
3. **Управление → Настройки → Работа с API → Callback API**:
   - Версия API: **5.199** (или совместимая).
   - Включите типы событий: **Входящее сообщение** (`message_new`), **Действие с сообщением** (`message_event`).
   - Укажите URL: `https://ваш-домен/vk` (HTTPS обязателен для продакшена; для разработки используйте ngrok/cloudflared).
   - **Важно:** путь должен быть именно **`/vk`** (или **`/vk/`**). Без этого сегмента ВК получит **404**.
   - Скопируйте строку подтверждения в `VK_CONFIRMATION`.
   - Если в Callback задан **секретный ключ**, в каждом JSON (включая `confirmation`) ВК передаёт поле `secret` — значение должно **совпадать** с `VK_CALLBACK_SECRET` в `.env`. Иначе ответ будет **403** и проверка не пройдёт.
4. Добавьте бота-сообщество в **беседу водителей** (как участника с доступом к сообщениям). Узнайте `peer_id` беседы: проще всего временно залогировать входящее `message_new` из этой беседы и скопировать `peer_id` в `DRIVERS_PEER_ID`.

## Запуск

```bash
cp .env.example .env
# заполните .env
npm install
npm start
```

Переменные см. в `.env.example`.

### Цены

Отредактируйте `prices.txt` в корне проекта — бот читает файл при каждом нажатии **«💰 Цены»** (перезапуск не нужен).

### Админ

1. Узнайте свой VK `user_id` (из ссылки профиля или через API).
2. Добавьте в `.env`: `ADMIN_VK_IDS=ваш_id` (несколько — через запятую).
3. В ЛС с ботом: **«⚙️ Админ»** — список заявок водителей, смена `prices.txt` из диалога.

## Проверка

- `GET /health` → `ok`
- После настройки Callback ВК отправит **POST** с JSON `type: confirmation` — сервер отвечает **HTTP 200** и телом **`text/plain`**: строка из `VK_CONFIRMATION` (например `a6100b7e`).
- Если открыть в браузере `https://…/vk` — будет **GET** и подсказка-текст (это не ошибка; ВК всё равно шлёт **POST**).

### Если ВК пишет про 404

1. В поле URL в Callback указан **тот же хост**, что выдаёт туннель, и путь **`/vk`** в конце.
2. В момент нажатия «Подтвердить» запущены **`npm start`** и туннель (ngrok/cloudflared).
3. Проверьте с компьютера:  
   `curl -X POST 'https://ВАШ_ТУННЕЛЬ/vk' -H 'Content-Type: application/json' -d '{"type":"confirmation","group_id":ВАШ_ID}'`  
   В ответе должна быть **одна строка** подтверждения. Если в ВК включён секрет — добавьте в JSON поле `"secret":"…"` как в настройках.

### ЛС с сообществом и `peer_id`

В Callback для диалога с сообществом ВК может прислать `peer_id` как **`2000000000 + user_id`**, так и **`peer_id === user_id`**. Бот учитывает оба варианта и отвечает в **`messages.send`** тем же `peer_id`, с которого пришло сообщение.

### Интерфейс на кнопках

**Пассажир (ЛС):** Заказать / Цены / Я водитель / Помощь.  
**Водитель (одобрен):** Профиль / Цены / Помощь; в поездке — Завершить / Помощь.  
**Водитель (ожидает):** Профиль / Цены / Помощь — заказы в беседе недоступны до одобрения.  
**Водители (беседа):** постоянно Помощь; на новом заказе — ~3-5 / ~10 / ~20 мин (далее — только правка поста «кто взял»).
